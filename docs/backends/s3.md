# S3 Backend

Ghostsnap supports native S3-backed repositories.

## Current Status

- `s3:bucket/prefix` repository URIs are supported end-to-end
- `ghostsnap init --backend s3 ...` initializes an S3-backed repository
- All commands (`backup`, `restore`, `snapshots`, `check`, `stats`, `ls`, `dump`, `diff`, `copy`) work with S3 repositories
- S3-compatible providers (MinIO, Wasabi, DigitalOcean Spaces, etc.) work via endpoint override

## Usage

```bash
# Initialize an S3-backed repository
ghostsnap init --backend s3 --bucket my-bucket --prefix backups s3:my-bucket/backups

# Back up to S3
ghostsnap --repo s3:my-bucket/backups backup /data

# List snapshots
ghostsnap --repo s3:my-bucket/backups snapshots

# Restore from S3
ghostsnap --repo s3:my-bucket/backups restore <snapshot-id> --target /restore
```

## S3-Compatible Storage

Use the `--endpoint` flag for S3-compatible providers:

```bash
# MinIO
ghostsnap init --backend s3 --bucket backups --endpoint http://localhost:9000 s3:backups

# Wasabi
ghostsnap init --backend s3 --bucket my-bucket --endpoint https://s3.wasabisys.com s3:my-bucket

# Backblaze B2
ghostsnap init --backend s3 --bucket my-bucket --endpoint https://s3.us-west-004.backblazeb2.com s3:my-bucket

# DigitalOcean Spaces
ghostsnap init --backend s3 --bucket my-bucket --endpoint https://nyc3.digitaloceanspaces.com s3:my-bucket
```

### Provider-Specific Notes

**Backblaze B2**: Use the S3-compatible API endpoint for your region. Find the endpoint in your B2 bucket settings under "Endpoint". Ghostsnap also provides a `b2:` URI scheme; see the [B2 backend](b2.md) documentation.

**MinIO**: Use your MinIO server's URL. Default is `http://localhost:9000` for local development. Ghostsnap also provides a `minio:` URI scheme; see the [MinIO backend](minio.md) documentation.

#### Wasabi

Wasabi is S3-compatible. Select the endpoint for your bucket's region:

- US East 1: `https://s3.wasabisys.com` (also `https://s3.us-east-1.wasabisys.com`)
- US East 2: `https://s3.us-east-2.wasabisys.com`
- EU Central 1: `https://s3.eu-central-1.wasabisys.com`

Use the standard `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` credentials.

```bash
ghostsnap init --backend s3 \
  --bucket mybucket \
  --endpoint https://s3.wasabisys.com \
  s3:mybucket/backups
```

## Configuration

### Environment Variables

```bash
export AWS_ACCESS_KEY_ID="AKIAIOSFODNN7EXAMPLE"
export AWS_SECRET_ACCESS_KEY="wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
export AWS_REGION="us-west-2"
```

### Server-Side Encryption

The init command accepts S3 server-side encryption flags:

```bash
# SSE-S3 (AES256)
ghostsnap init --backend s3 --bucket my-bucket --sse-type aes256 s3:my-bucket/backups

# SSE-KMS
ghostsnap init --backend s3 --bucket my-bucket --sse-type kms --sse-kms-key-id alias/my-key s3:my-bucket/backups
```
