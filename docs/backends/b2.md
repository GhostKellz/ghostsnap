# Backblaze B2 Backend

Backblaze B2 is supported via S3 compatibility mode. The `b2:` URI scheme maps
internally to Ghostsnap's S3 backend, providing full repository support.

## Quick Start

The preferred form uses the `b2:` scheme with the endpoint supplied via
`B2_S3_ENDPOINT`. The backend is inferred from the URI scheme, so `--backend` is
not required.

```bash
# Set the B2 endpoint and credentials (S3-compatible keys)
export B2_S3_ENDPOINT="https://s3.us-west-004.backblazeb2.com"
export AWS_ACCESS_KEY_ID="your-b2-application-key-id"
export AWS_SECRET_ACCESS_KEY="your-b2-application-key"

# Initialize repository (scheme inferred from the URI)
ghostsnap init b2:my-bucket/backups

# Back up data
ghostsnap --repo b2:my-bucket/backups backup /data

# List snapshots
ghostsnap --repo b2:my-bucket/backups snapshots

# Restore
ghostsnap --repo b2:my-bucket/backups restore abc123 --target /restore
```

The `b2://my-bucket/backups` form is also accepted. The endpoint is resolved
from `B2_S3_ENDPOINT`, falling back to `AWS_ENDPOINT_URL`. The region is taken
from `AWS_REGION`, otherwise inferred from the endpoint host pattern
`s3.<region>.backblazeb2.com`.

### Alternative: Explicit S3 Backend

The older explicit S3 form still works and is equivalent:

```bash
export AWS_ACCESS_KEY_ID="your-b2-application-key-id"
export AWS_SECRET_ACCESS_KEY="your-b2-application-key"

ghostsnap init --backend s3 \
  --bucket my-bucket \
  --endpoint https://s3.us-west-004.backblazeb2.com \
  s3:my-bucket/backups

ghostsnap --repo s3:my-bucket/backups backup /data
```

## Setup

### 1. Enable S3-Compatible API

In your B2 account:
1. Go to "App Keys"
2. Create a new application key
3. Note the `keyID` (use as AWS_ACCESS_KEY_ID)
4. Note the `applicationKey` (use as AWS_SECRET_ACCESS_KEY)

### 2. Find Your Endpoint

Your endpoint depends on the bucket's region. Find it in your bucket settings:
- US West: `s3.us-west-004.backblazeb2.com`
- US East: `s3.us-east-005.backblazeb2.com`
- EU Central: `s3.eu-central-003.backblazeb2.com`

### 3. Configure Environment

```bash
export B2_S3_ENDPOINT="https://s3.us-west-004.backblazeb2.com"
export AWS_ACCESS_KEY_ID="your-b2-application-key-id"
export AWS_SECRET_ACCESS_KEY="your-b2-application-key"
```

| Variable | Description |
|----------|-------------|
| `B2_S3_ENDPOINT` | B2 S3-compatible endpoint. Falls back to `AWS_ENDPOINT_URL`. |
| `AWS_ACCESS_KEY_ID` | B2 application key ID (`keyID`). |
| `AWS_SECRET_ACCESS_KEY` | B2 application key (`applicationKey`). |
| `AWS_REGION` | Optional. Inferred from the endpoint host when unset. |

## Usage

All standard Ghostsnap commands work with B2 via the S3 protocol:

```bash
# Initialize
ghostsnap init --backend s3 --bucket my-bucket --endpoint https://s3.us-west-004.backblazeb2.com s3:my-bucket

# Backup
ghostsnap --repo s3:my-bucket backup /data --tag daily

# Check integrity
ghostsnap --repo s3:my-bucket check

# Stats
ghostsnap --repo s3:my-bucket stats

# Forget old snapshots
ghostsnap --repo s3:my-bucket forget --keep-daily 7 --keep-weekly 4

# Prune unused data
ghostsnap --repo s3:my-bucket prune
```

## Cost Optimization

Backblaze B2 pricing is straightforward:
- Storage: $0.006/GB/month
- Downloads: $0.01/GB (first 1GB free daily)
- API calls: Free for Class B, $0.004/10k for Class C

Tips:
1. Use `forget` and `prune` to remove old snapshots
2. Monitor storage with `ghostsnap stats`
3. B2's free egress tier works well for occasional restores
